# metrics/ — Layer ⑤ Measurement (optional)

> **Self-architecture role**: this directory is **Layer ⑤ Measurement** in pentaglyph's [self-architecture](../arc42/05-building-blocks/pentaglyph-self-architecture.md). It quantifies the health of layers ⓪-④. Layer ⑤ is **optional** — small projects may skip it without losing core kit value. See [ADR-0001](../arc42/09-decisions/0001-adopt-five-layer-self-architecture.md) for the optional-layer rationale and [ADR-0009](../arc42/09-decisions/0009-measurement-layer-activation.md) for the activation contract.

## What lives here

| File | Purpose |
| --- | --- |
| [`baseline.md`](./baseline.md) | Snapshot of pentaglyph's own (dogfooded) metric values at a point in time |
| [`snapshots/YYYY-MM-DD_*.md`](./snapshots/) | Periodic dated snapshots over time (append-only) |

Snapshots are append-only. Each snapshot is generated by running the [`scripts/docs/`](../../../scripts/docs/) metric scripts against the project's `docs/` tree and committing the markdown output.

## Metric categories (5)

| # | Category | Script | What it measures |
| --- | --- | --- | --- |
| 1 | **Coverage** | [`metrics_coverage.py`](../../../scripts/docs/metrics_coverage.py) | arc42 + detailed-design + use-cases / PRDs existence + substantive-content coverage |
| 2 | **Freshness** | [`metrics_freshness.py`](../../../scripts/docs/metrics_freshness.py) | Durable-doc `last-reviewed` age distribution + 90-day staleness warnings |
| 3 | **ADR throughput** | [`metrics_adr.py`](../../../scripts/docs/metrics_adr.py) | ADR Status distribution + monthly throughput + stale-Proposed flag |
| 4 | **Doc rot detection** *(forthcoming)* | TBD | Cross-reference broken-link rate + spec/code divergence heuristics |
| 5 | **Adoption** *(forthcoming)* | TBD | (For pentaglyph upstream only) Public downstream-project count + binding diversity |

Categories 4 and 5 are documented in [ADR-0009](../arc42/09-decisions/0009-measurement-layer-activation.md) as future expansion. Categories 1-3 are usable today.

## Running the scripts

### Direct (Python 3.10+, stdlib-only)

```bash
# Coverage
python3 libs/pentaglyph-docs/scripts/docs/metrics_coverage.py docs/

# Freshness (default staleness threshold 90 days; override with --stale-days=N)
python3 libs/pentaglyph-docs/scripts/docs/metrics_freshness.py docs/

# ADR throughput
python3 libs/pentaglyph-docs/scripts/docs/metrics_adr.py docs/

# JSON output for CI ingestion
python3 libs/pentaglyph-docs/scripts/docs/metrics_coverage.py docs/ --format=json
```

### Via Bun CLI

```bash
bunx pentaglyph metrics --target=docs                           # all 3, markdown
bunx pentaglyph metrics --target=docs --metric=coverage --format=json
```

The CLI ([`../../../cli/src/commands/metrics.ts`](../../../cli/src/commands/metrics.ts)) is a thin wrapper that shells out to the Python scripts.

## Capturing a snapshot

To save a dated snapshot for trend tracking:

```bash
DATE=$(date -u +%Y-%m-%d)
mkdir -p docs/metrics/snapshots
bunx pentaglyph metrics --target=docs > docs/metrics/snapshots/${DATE}_full.md
git add docs/metrics/snapshots/${DATE}_full.md
git commit -m "chore(metrics): snapshot ${DATE}"
```

## How to interpret

### Coverage

- **Substantive threshold**: a file with ≥ 200 characters of body text (after stripping YAML front-matter and Markdown heading lines) is "substantive". Below that, it is a stub.
- **Coverage %**: substantive files / total files per arc42 section.
- **Missing sections**: arc42 sections expected by the kit that have no directory yet. These should be created (even if empty README) per arc42 spec.
- **Target**: 90%+ overall coverage for v1.0 release; missing sections should be 0 (every arc42 section has at least a stub README).

### Freshness

- **Buckets**: fresh <30d / aging 30-89d / stale 90-179d / rotten 180d+ / missing `last-reviewed`.
- **Target**: < 10% stale, < 1% rotten, < 5% missing for healthy living-documentation discipline.
- **Action on stale**: review the doc, decide whether it is still accurate; if accurate, refresh `last-reviewed:`; if outdated, fix or supersede.

### ADR throughput

- **Status distribution**: healthy projects show a long tail of Accepted with occasional Proposed (in-flight) and rare Superseded (decisions explicitly revised).
- **Monthly throughput**: 1-5 ADRs/month for a small team is typical. Zero ADRs across multiple months suggests decisions are happening without record (Layer ④ Governance failure).
- **Stale Proposed**: per [`governance/adr-accept-protocol.md`](../governance/adr-accept-protocol.md), indefinite Proposed is forbidden. Proposed ADRs > 30 days should be accepted, rejected, or deprecated.

## Layer-writes contract

Per [ADR-0007](../arc42/09-decisions/0007-automation-layer-contract.md) §3, Layer ③ Automation scripts (the metric scripts in `scripts/docs/`) **write only to stdout** — the calling user redirects to Layer ⑤ Measurement files (this directory). The scripts never mutate `docs/` directly.

## Why is Layer ⑤ optional?

Per [ADR-0001](../arc42/09-decisions/0001-adopt-five-layer-self-architecture.md) §"Decision Outcome":

- Small / early-stage projects do not yet have enough docs to make metrics meaningful (a project with 5 docs total cannot have a useful coverage percentage).
- Some adopters explicitly do not want CI to fail on doc-freshness lint.
- Layer ⑤ requires running scripts as part of CI, which has a non-trivial setup cost.

[ADR-0009](../arc42/09-decisions/0009-measurement-layer-activation.md) describes when to activate Layer ⑤.
